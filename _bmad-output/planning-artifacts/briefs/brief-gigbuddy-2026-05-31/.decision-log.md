# Decision Log — GigBuddy Product Brief

## Session: 2026-05-31

### Discovery — captured from analyst conversation

- **Solo tool, not collaborative.** Sandy is the only user. No bandmate access required. Future productisation is a distant possibility, not a design driver.
- **Primary device for performance mode: iPhone** (sits on top of Nord keyboard during gigs). Notes are read between songs, not during.
- **Chord representation: text-based symbols** only. No PDFs, images, or iReal Pro-style charts.
- **Jazz band (primary): ~30-40 songs.** Covers band is secondary.
- **Song library built manually.** Migration from Apple Notes not required — Sandy will add songs organically, driven by setlist preparation workflow.
- **Setlist ingestion workflow:** Paste raw WhatsApp setlist text → app identifies unknown songs → Sandy adds notes for each. This is the primary library growth mechanism.
- **Setlist format observed:** Plain text, song per line, section headers (1st Set / 2nd Set / Reserve), occasional parenthetical annotations (e.g. "(vocal?)") that are per-gig notes, not permanent song attributes.
- **Multi-band support required from day one, architected as N bands.** Three bands confirmed: jazz (primary), covers (secondary), originals (infrequent). Data model must treat "band" as a first-class entity, not hardcoded to two.
- **Product name confirmed: GigBuddy.**
- **Multi-band is N bands, not hardcoded to two.** Third band confirmed: originals (infrequent). Data model must treat "band" as a first-class entity.
- **Brief finalised 2026-05-31.** Structural pass: merged Vision into V2 Horizon, moved Design Principles above The Problem, condensed Who This Serves, fixed three-band reference in The Solution. Prose pass: fixed fragment in Who This Serves, added missing comma in V2 Horizon.
- **Hosted on Sandy's own AWS account.** Web app (MacBook) + PWA (iPhone).
- **Performance mode:** streamlined card-style view — one song at a time, large clear text, fast navigation.
- **Practice mode:** richer content — external links, learning notes, extended chord charts.
- **Covers band needs fuller chord charts** during performance; jazz band is more improvisation-focused with briefer in-gig prompts.
