# Real example — Big Ed's wedding gig notes

Sandy's actual notes for a real gig, pasted verbatim below. This is what the current pre-gig "compile notes document" workflow produces — the document he opens on his iPhone on top of the Nord during the gig.

**Signals to preserve / honour:**

- **Capitalisation is meaningful.** ALL CAPS song titles = confirmed in set. Mixed case (e.g. "Cissy Strut", "At the Midnight Hour") = optional / TBA / dependent on running time. The original document used visual highlighting for the same signal — capitalisation is the surviving cue here.
- **Patches are referenced by number** with letter prefixes (P11, P13, P14, P31, P33, P34, R41). The R prefix appears on at least one (R41 = Piano and Cello) — naming convention is the user's own.
- **Per-song instrument switches are common** ("Switch to Rhodes and play long"). One song may use multiple sounds.
- **Multiple-key support is real:** "F#m(Ian) Fm(Sandy)" — same song, different key depending on who's singing.
- **Alternate keys for soloists:** "[for guitar solo]" block with transposed changes.
- **Transpose flags:** "TRANSPOSE?? -2 = Cm/F", "Or transpose -1", "(P33 IS TRANSPOSED!)" — the Nord transpose setting is itself a piece of performance information.
- **SOLO marker** on some titles ("SUNNY - Bobby Hebb - SOLO", "CANTALOUPE ISLAND – SOLO") signals a feature spot.
- **Per-gig context** baked in: "INTO THE MYSTIC [first dance]".
- **Chord notation is loose** and mixes formal (Fm Ab7 Dbmaj7), shorthand ("Gm blues", "C blues"), prose ("descending line F, E, Eb, D"), and rhythmic hints ("D A C D, D A C").
- **Set separator** is an ASCII rule (`----`). Section names are "Set 1" / "Set 2".
- **Inline performance reminders** mixed with chord info: "Remember prominent piano in gap towards end", "HORNS in with theme", "After initial solos, stick on Fm for jam".
