## 2026-06-09 — §4.3 walkthrough

- **FR-21 End-of-setlist treatment resolved.** `NEXT ›` becomes inert (disabled) on the last Song. No `End performance ›` button. Rationale per Sandy: prevents accidental termination via the same gesture used to advance Songs (i.e., last-song NEXT tap would otherwise be hazardous). Updated FR-21, §8 Open Questions item 3 (marked resolved), and Assumptions Index (entry removed).
- **FR-18 Wake Lock fallback resolved.** Best-effort acquisition + reacquisition on foreground transition + reacquisition on detected release. When Wake Lock is not held, a persistent static state indicator surfaces on the Performance Card. User recovery is screen tap. Reasoning: my original "silent + recover-by-tap" recommendation traded user awareness for a rule that wasn't being violated (banned toasts/modals are transient; a persistent state indicator is not). Sandy pushed back on the recommendation; refined to (d). Updated FR-18 description, consequences, and NFRs; marked §8.4 resolved.

Removed `[ASSUMPTION]` and `[NOTE FOR PM]` tags; cleaned up Assumptions Index. §8.1 retains as "fully deferred to Architecture" — still an open question, but with no PM ask attached.

## 2026-06-09 — §4.7 walkthrough

## 2026-06-09 — §6.2 sharpening

## 2026-06-09 — §7 Success Metrics

- **SM-2 rewritten qualitative.** Dropped my invented 10-minute target — gameable and not user-validated. New formulation tests for "fluency, not speed": Sandy can process a ~20-song Setlist with Unknowns end-to-end without context-switching out of the prep flow.
- **Added SM-C3 (don't optimize offline at cost of online responsiveness).** Real risk given §4.7 offline emphasis; primary usage is online prep.
- **Added SM-C4 (don't add settings/toggles to satisfy edge cases).** Aligns with the no-Settings-surface decision and the brief's "personal tool, not a platform" principle.

## 2026-06-09 — §8.10 AWS cost

- **No fixed monthly cap.** Architecture should flag any choice materially more expensive than the cheapest viable alternative; non-obvious spend requires a cost-justification statement in the architecture document. Updated §8.10 accordingly.

## 2026-06-09 — §A walkthrough

- **§A.2 availability target dropped.** 99% rolling-30-day was invented and didn't capture the real requirement. Replaced with qualitative contract: must be available during scheduled gig windows; routine maintenance must avoid weekend evenings (Friday–Sunday 18:00–24:00 local); best-effort otherwise; no numeric SLO. `[ASSUMPTION]` tag removed.
- **§A.4 no-analytics confirmed.** Setlist history (already first-class per §4.5 V2-readiness) implicitly captures usage data; CloudWatch covers diagnostic layer. No additional telemetry added. Aligns with brief's "personal tool, not a platform" and SM-C4.

## 2026-06-09 — Finalize input reconciliation

Three subagents read brief / DESIGN.md / EXPERIENCE.md against the PRD. Eight actionable gaps surfaced and applied:

1. **FR-30 toast suppression in Performance Mode.** Added consequence: toasts suppressed during Performance Mode; save-failure toasts held and surfaced on exit.
2. **§5 pinch/multi-finger gesture ban.** Added Non-Goal: no multi-finger gestures anywhere; pinch-zoom suppressed.
3. **Anti-toggle framing in §1 Vision.** Added paragraph: Practice and Performance are different surfaces sharing data, not view modes. Implementations must not introduce a view-mode toggle.
4. **FR-29 setlist history first-class.** Added consequence: every Setlist persists in full from day one so V2 analytics require no migration.
5. **FR-5 'not the same content filtered' principle.** Expanded Notes to make explicit that Performance/Practice fields are distinct content, not display rules over one document.
6. **§4.3 20–30s read budget.** Added paragraph naming the load-bearing time budget that justifies type floors and single-tap nav.
7. **§A.1 'latency is a defect' framing.** Added header sentence: Performance Mode latency miss is a bug class, not polish; Practice Mode held to ordinary standards.
8. **§5 theme toggle Non-Goal.** Added: no user-facing theme toggle in V1; light/dark are surface defaults, not user-selectable variants.

Two further gaps dropped:
- §1 raison d'être sharpening (style, not substance).
- Per-gig annotation absent → row collapse (already covered by EXPERIENCE.md; over-specification).

Reconciliation files persisted at `reconcile-brief.md`, `reconcile-design.md`, `reconcile-experience.md`.
